# v1.0.0 — Pentest-Ready Insecure MVP

Tagged from v0.9.5-rc.1 per [STRATEGY.md](../roadmap/STRATEGY.md). Tag unchanged; post-tag fixes land on `main`.

## What's included

- Full auth, files, sharing, ternary RBAC, admin polish
- Pagination, validation, unified errors, request logging
- Persistent audit logs, admin search/stats, health endpoint
- Docker production stack (`infra/docker-compose.prod.yml`)

## Post–v1.0.0 tag fixes on `main`

### v0.9 product UX

- Product UI for user/moderator/admin; API explorers at `/dev`
- Demo user seed migration (ADR-029)
- Role-aware nav, moderator review queue, sharing file picker

### Pre-pentest polish (v1.0.x readiness)

- Per-user file/share filtering in product UI; API IDOR unchanged
- Demo files/shares seed + repair migrations (ADR-030)
- Sharing bug fixes: `publicToken` on PUT, orphan cleanup on file delete
- `/share/[token]` friendly landing; product UI shows friendly URLs
- Dead routes removed (`/admin/[id]`); explorers consolidated under `/dev/*`
- [v1.0.0-ground-truth.md](../security/Cycle-1/Dev/v1.0.0-ground-truth.md) + [v1.0.0-pentest-ready.md](v1.0.0-pentest-ready.md)
- 150 e2e tests; extended `journey-test.sh`

### Docker contract fixes

These commits close Docker/browser contract gaps without moving the `v1.0.0` tag:

- Frontend `NEXT_PUBLIC_API_URL=/api` for nginx proxy at `:8080`
- Paginated list unwrap for `GET /files`, `/sharing`, `/users`
- Separate `pgdata_prod` volume from dev `pgdata`
- Expanded `infra/smoke-test.sh` (health → register → upload → list)
- `infra/e2e-docker.sh` — 150 e2e tests against Docker `kc_prod`
- nginx 413 (1m body limit) documented as accidental infra ceiling
- Home page version string updated to v1.0.0 baseline

## Known intentional weaknesses

See [cwe-inventory.md](../security/cwe-inventory.md). **59 documented instances across 38 unique CWE IDs.**

## Deploy

```bash
docker compose -f infra/docker-compose.prod.yml up -d --build
./infra/smoke-test.sh
./infra/e2e-docker.sh   # optional full e2e against Docker postgres
```

Browser: `http://localhost:8080`
